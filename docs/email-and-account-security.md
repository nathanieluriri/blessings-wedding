# Email, Notifications & Account Security

How the dashboard sends email and protects admin accounts. All four features
share one provider (Resend) and one rule: **secrets and temporary passwords are
generated by the system, never typed by another admin.**

## Why Resend

Every reputable transactional-email provider (Resend, Postmark, Brevo, SendGrid,
SES…) requires SPF + DKIM DNS records to deliver to arbitrary recipients — it's
anti-abuse, not provider-specific. Given that, Resend is the lightest path:

- **Code = the API key.** `resend.emails.send({ from, to, subject, html, text })`.
  No SMTP, no webhooks, no template service. Plain HTML templates live in
  [lib/email/templates.ts](../lib/email/templates.ts).
- **One-time DNS** to email anyone; until then it delivers only to your own
  Resend account email (test mode).
- Free tier: 100 emails/day, 3,000/month.

## Setup

Set these in `.env.local` (see [.env.example](../.env.example)):

```
RESEND_API_KEY=re_...
RESEND_FROM="Blessing & Justice <noreply@yourdomain.com>"
```

To email recipients other than your own account address:

1. In the Resend dashboard → **Domains** → add your domain.
2. Add the two TXT records it shows (SPF + DKIM) at your DNS host.
3. Wait for **Verified** (usually minutes), then point `RESEND_FROM` at that domain.

Email **degrades gracefully**: with no `RESEND_API_KEY`, sends are skipped with a
console warning and nothing throws — local dev still runs.

## 1. New-RSVP notifications

When a guest submits the public form, [app/api/rsvp/route.ts](../app/api/rsvp/route.ts)
fires `after(() => sendNewRsvpNotification(...))` — email runs *after* the
response, so it can never block or break a submission.

Recipients = **every admin's email + a custom list**, managed at **Admin →
Notifications** ([app/admin/notifications](../app/admin/notifications)). Toggle the
whole feature on/off and send a test email there. Settings live in the `settings`
collection under `_id: "notifications"`.

## 2. Admin invites (system-generated password)

**Rule: the inviting admin never sees or sets the new admin's password.**

At **Admin → Admins**, "Add admin" collects only email/name/role.
[app/api/admin/admins/route.ts](../app/api/admin/admins/route.ts) generates a
temporary password, stores only its bcrypt hash, and emails it to the invitee
with a sign-in link. The invitee is forced to set their own password on first
login. If the invite email fails to send, the account is rolled back. A
**Resend invite** action rotates the password and re-emails it (also
system-generated).

## 3. Two-factor authentication (TOTP + email fallback)

Opt-in per admin at **Admin → Security** ([app/admin/security](../app/admin/security)).

- **Primary:** authenticator app (TOTP). Enrollment shows a QR code; the secret
  is stored **AES-256-GCM-encrypted** (key derived from `SESSION_SECRET`, so
  there's no extra secret to manage — see [lib/auth/crypto.ts](../lib/auth/crypto.ts)).
- **Recovery codes:** 10 single-use codes shown once at enrollment, stored
  bcrypt-hashed.
- **Email fallback:** "Email me a code" sends a 6-digit code (10-min expiry,
  attempt-capped) for when the authenticator isn't available.

**Login is two-step** when 2FA is on: a correct password issues a short-lived
*pending* cookie (`admin_2fa`) that grants no access on its own; a verified code
at [/api/admin/auth/2fa/verify](../app/api/admin/auth/2fa/verify/route.ts) swaps
it for a real session. `proxy.ts` only honors the real session cookie.

## 4. Forgot / reset password

"Forgot password?" on the login screen → [/login/admin/forgot](../app/login/admin/forgot).
[forgot route](../app/api/admin/auth/forgot/route.ts) always responds the same
way (no account-enumeration); if the email is an admin, it stores a sha256-hashed,
45-minute, single-use token and emails a reset link to
[/login/admin/reset](../app/login/admin/reset). Reset sets the new password but
does **not** sign you in — you then log in normally, so a reset can never bypass
2FA.

## Data & expiry

- `admins` — extended with `twoFactorEnabled`, `totpSecretEnc`, `totpPendingEnc`,
  `recoveryCodeHashes`.
- `settings` — adds the `_id: "notifications"` document.
- `passwordResets`, `otpChallenges` — new collections with **TTL indexes**
  (`expireAfterSeconds: 0` on `expiresAt`), so expired tokens/codes self-delete.
